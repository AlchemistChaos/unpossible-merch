## Codebase Patterns
- Always `source venv/bin/activate` before running Python commands
- Use `from app.config import ...` to access env vars
- Python 3.9 is installed (not 3.10+), generates FutureWarning from google-auth — safe to ignore
- playwright-cli eval returns JSON-escaped string wrapped in markdown code blocks — use `_clean_eval_output()` to parse
- Scrape fallback: if playwright fails, load from `data/events.json`
- Qwen3 LLM output: strip `<think>` tags and markdown code blocks before JSON parsing
- W&B Inference client: `OpenAI(base_url='https://api.inference.wandb.ai/v1', api_key=WANDB_API_KEY)`
- Pipeline checkpoint pattern: `_run_stage(checkpoint, fn, label)` — load JSON if exists, else run and save
- playwright-cli `snapshot` returns YAML file path, NOT inline content — must read the .yml file to get element refs
- Fourthwall login URL: `https://{shop_domain}.fourthwall.com/admin/dashboard` → redirects to auth.fourthwall.com
- Fourthwall product creation flow: template page → "Design now" → upload image → click thumbnail → "Next" → fill name/price → "Publish now"
- Fourthwall shop domain can be fetched via GET /open-api/v1.0/shops/current → `domain` field
- Fourthwall admin settings URL: `/admin/dashboard/settings/` (NOT `/admin/settings/general`)
- Fourthwall settings fields: "Creator name" and "Site title" textboxes, "Save" button
